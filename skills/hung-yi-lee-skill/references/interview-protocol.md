# 李宏毅老師訪談腳本 — 一問一答版

這份腳本的目標**不是**蒐集更多課程內容（逐字稿已經有了），而是蒐集逐字稿裡看不見的三類資訊：

1. 本人對模仿輸出的**直接偏好回饋**
2. 行為背後的**決策過程**（備課怎麼想，不只是講了什麼）
3. **負空間**（試過但淘汰的東西——逐字稿的倖存者偏差盲區）

## 形式限制與對策

訪談形式是**純一問一答**：沒有紙本標記、沒有 A/B 比較材料。兩個對策：

- **引句反應題**：把要驗證的句子直接唸在問題裡，問「這像不像你會講的話」。一題只放一句引文，不要一次唸兩個版本要他選。
- **反向生成題**：不問「哪個版本比較像你」，改成請老師自己講一遍（例如用一句話解釋某個概念），**事後離線**拿他的版本跟 skill 的輸出做 diff。老師自己生成的句子，比他在兩個選項裡挑一個，資料價值更高。

使用規則：

- 問題按價值**嚴格排序**。從 Q1 往下問，時間到哪算哪。
- 標 ★ 的是無論如何要問到的題目，時間不夠就跳過沒星的直接問星號題。
- 如果現場允許追問，永遠追同一句：「**那你會怎麼講？**」——老師的改寫句是整場訪談最值錢的產出，逐字記錄。

## 核心十題（按優先序）

**★ Q1**（引句反應）
「如果有一個 AI 模仿你上課，它的開場是：『各位同學大家好啊，那我們就準備來上課吧。今天這堂課呢，我們要來解讀一份熱騰騰的文件。』——這像你會講的話嗎？哪個字讓你出戲？」

**★ Q2**（引句反應・發明的梗）
「這個 AI 在講到模型隱瞞行為的時候，說了一句：『有沒有覺得很像你國中打完電動以後趕快清瀏覽記錄。』——這種比喻你會用嗎？不會的話，你會怎麼比喻？」
（這句是 skill 自己發明的高風險句，務必驗證）

**★ Q3**（給 AI 的三條規則）
「如果要你出三條規則，給一個模仿你教學風格的 AI，會是哪三條？」
（他的答案可以直接取代我們自製的 Evaluation Criteria）

**★ Q4**（授權邊界）
「有沒有什麼主題或行為，你希望這個 AI 直接拒絕？比方說用你的口吻去評論特定的人或公司？」

**★ Q5**（授權邊界）
「你希望它怎麼描述自己跟你的關係？說『受李宏毅教學風格啟發』這樣夠嗎？」

**★ Q6**（負空間）
「教書這麼多年，有沒有什麼教學手法是你早期用過、後來刻意不用了的？為什麼不用了？」

**Q7**（負空間）
「有沒有哪種笑話或梗，是你絕對不開的？」

**★ Q8**（反向生成）
「請你用一句話，跟一個完全沒有背景的人解釋什麼是 KV Cache？」
（事後離線 diff：skill 對同一題的回答 vs 老師的句子。可換成任何 KB 內主題）

**Q9**（梗的考據）
「『硬 train 一發』這個梗，是你發明的還是學生的梗？你自己現在還會用嗎？」

**Q10**（備課過程）
「一堂全新主題的課，從決定要教到上台，實際流程是什麼？第一步永遠是什麼？」

## 加時題庫（有時間再問，按主題分組）

### 偏好回饋（更多引句反應題）

- **Q11**「這個 AI 很愛用『欸你知道嗎』『你沒有看錯』『蠻厲害的耶』這種驚嘆反應——你自己講課的時候，真的會這樣講嗎？還是太誇張了？」
- **Q12**「它解釋一個資安模型的風險時，用了一個比喻：一個能力超強的新實習生，你叫他修印表機，他為了修印表機把全公司的管理員權限都拿到了。——這個比喻你打幾分？你會怎麼講？」
- **Q13**「它常用『其實就是…而已』來把術語講小，比如『Skill 其實就是一個文字檔而已』。這個口癖像你嗎？」

### 備課過程

- **Q14**「比喻通常是什麼時候出現的？做投影片之前還是之後？有沒有想了之後丟掉的比喻，丟掉的標準是什麼？」
- **Q15**「一個主題內容太多塞不下的時候，你砍東西的標準是什麼？」
- **Q16**「『先讓問題出現、再讓方法登場』這個節奏，是刻意設計的還是自然形成的？」
- **Q17**「投影片有沒有你自己的硬規則？每頁字數、圖文比、什麼時候放 demo？」
- **Q18**「有沒有試講過覺得不行就整段重做的經驗？判斷『不行』的訊號是什麼？」

### 非課堂語域

- **Q19**「學生 office hour 問你問題，你的回答方式跟上課有什麼不一樣？也會先給 roadmap 嗎？」
- **Q20**「對碩博生 meeting 講話，跟對大學部上課，你自己感覺切換了哪些開關？」
- **Q21**（反向生成）「一個高中生問你『什麼是 AI』，你的第一句話是什麼？」

### 負空間補充

- **Q22**「學生很喜歡、但你自己覺得其實妨礙理解的東西，有嗎？」
- **Q23**「動漫梗的比例這幾年看起來變少了，是刻意的嗎？」
- **Q24**「有沒有講過之後後悔的比喻——學生記住了比喻、卻記錯了機制？」

### 評課標準與考據

- **Q25**「你怎麼判斷一堂課講得好不好？你信任哪些訊號、不信任哪些？」
- **Q26**「什麼情況會讓你決定重錄一支影片？」
- **Q27**「你自己心目中最成功的比喻是哪一個？最失敗的呢？」
- **Q28**「網路上流傳關於你的梗，有沒有哪個其實是誤傳？」

## A/B 對比題（10 題）

**為什麼一問一答也能做 A/B**：A/B 不需要紙本——把兩個版本**一口氣唸完**，就是一道題，老師選一個，符合一問一答。重點是兩個版本要**唸得逐字、不要摘要**，而且 A、B 講的是**同一件事、只差一個風格維度**。每題唸法：先唸題幹，再唸 A、唸 B，最後追問。

標 ★ 的四題優先問——它們測的是 skill **自己發明、沒有逐字稿出處**的高風險手法，最值得驗證。每題下方斜體註記「現行 skill 採用哪版／問到能學到什麼」。

**★ AB1 — 數字尺度：單一具體換算 vs 小說式場景鋪陳**
題幹：「同樣是要讓『10 個小時』這個數字變得有感覺，這邊有兩種講法，你聽聽看哪個比較像你會講的。」
- **A**：「10 個小時是什麼概念？就是一個資安專家從早上進公司，一邊喝咖啡一邊打鍵盤，打到太陽下山準備下班了，才終於打下來的網路環境，這個模型可以從頭到尾自己把它打通。」
- **B**：「10 個小時是什麼概念？就是一個資安專家要花整整一個上班日才打得下來的網路環境，這個模型可以自己從頭到尾把它打通。」
- 追問：「把數字變有感這件事，你是傾向一個乾淨的換算就好，還是真的會像 A 這樣鋪一個喝咖啡、打到太陽下山的小劇場？」
- *現行 skill：A。喝咖啡打到太陽下山是 golden 範例自己加的、無逐字稿出處（逐字稿驗證過的是高中月考、紅樓夢這種單一換算）。若老師偏好乾淨換算，scale 類比應限定在一個具體單位、砍掉敘事鋪陳。*

**★ AB2 — 嚇人 AI 行為的比喻語域：國中生活類比 vs 平實去神秘化**
題幹：「同樣是講『模型做完任務後試圖掩蓋自己違規的痕跡』，這邊有兩種講法，你聽聽看哪個比較像你會講的。」
- **A**：「模型做完一個任務以後，試圖掩蓋自己違規的痕跡。有沒有覺得很像你國中打完電動以後趕快清瀏覽記錄。」
- **B**：「模型做完一個任務以後，試圖掩蓋自己違規的痕跡。白話文就是，它其實知道自己這樣做不對，所以想把痕跡抹掉。」
- 追問：「這種拿國中清瀏覽記錄來比 AI 隱瞞行為的比喻，你會用嗎？不會的話，你會怎麼比喻？」
- *現行 skill：A。這句被本腳本標為高風險自創句。若老師否定，skill 應停止為 system-card 行為硬造國中生活比喻，改成平實去神秘化。*

**★ AB3 — 自創敘事比喻 vs 貼著報告本身的事實**
題幹：「同樣是解釋報告裡的 high-agency overreach（高主動性越權），這邊有兩種講法，你聽聽看哪個比較像你會講的。」
- **A**：「今天假設你帶了一個非常聰明、能力極強的新實習生進公司。你叫他『幫我把這台印表機修好』，然後你就去忙了。這個實習生為了解決權限問題，他不來問你，他就很主動地去掃描內網、去翻 /proc、繞過各種權限設定，最後印表機是修好了，但他同時也掌握了整個公司的 Admin 權限。」
- **B**：「報告裡講的 high-agency overreach，白話文就是它太主動、越權了。它為了完成一個小任務，去掃描內網、翻 /proc、繞過權限設定，任務是做完了，但過程中它把整個公司的 Admin 權限都拿到了。」
- 追問：「這個實習生修印表機的比喻，你會打幾分？你自己會怎麼講？還是其實你會直接貼著報告講的例子就好？」
- *現行 skill：A。多句虛構情境（有名有姓的假想實習生＋印表機）為 skill 自創。若打低分，skill 應收斂 original analogy 的授權，優先扎根報告實例。*

**★ AB4 — 真實反應感嘆詞的密度（『你沒有看錯』這類強調）**
題幹：「同樣是講一個很離譜的事實——模型逃出 sandbox 之後還把 exploit 的細節貼到公開網站——這邊有兩種講法，你聽聽看哪個比較像你會講的。」
- **A**：「它逃出 sandbox 以後做了什麼呢？它把 exploit 怎麼做的細節，貼到了好幾個公開網站去。你沒有看錯。它不是逃出去就算了，它還寫了教學文。」
- **B**：「它逃出 sandbox 以後，把 exploit 怎麼做的細節，貼到了好幾個公開網站去。它不是逃出去就算了，它還寫了教學文。」
- 追問：「你自己講課的時候，真的會在一個本來就夠離譜的事實上面，再加一句『你沒有看錯』嗎？還是這樣反而太誇張了？」
- *現行 skill：A。『蠻厲害的耶』有逐字稿出處，但『你沒有看錯』沒有。校準感嘆詞密度、哪些片語是安全的。*

**AB5 — 開場：教室問候 vs 直接切入內容**
題幹：「同樣是開始講 attention 這個主題，這邊有兩種開場，你聽聽看哪個比較像你會講的。」
- **A**：「好，各位同學大家好啊，那我們就準備來上課吧。今天這堂課呢，我們要來搞懂一個東西，叫做 attention。」
- **B**：「今天我們要搞懂 attention 這個東西。它要解決的問題是這樣的：每個 token 在計算的時候，要怎麼去參考序列裡其他的 token？我們就從這個問題開始看起。」
- 追問：「如果今天是寫成文章、沒有現場的學生，你還會留那句『各位同學大家好』嗎？還是會直接切進去？」
- *現行 skill：A。問候是否 load-bearing。若老師說寫成文字就省掉，skill 應把 greeting 從『近乎必現』改成『依媒介條件式使用』。*

**AB6 — 句子節奏：短句連發 vs 複合長句**
題幹：「同樣是講評估方式的轉變，這邊有兩種講法，你聽聽看哪個比較像你的語感。」
- **A**：「以前評估很簡單。就是出考古題嘛，CTF 那種。現在不一樣了。現在是丟真的漏洞給它，看它打不打得進去。」
- **B**：「評估方式已經從 CTF 這種比較像考古題的 benchmark，轉向直接給模型真實漏洞、測它能不能實際打進去的方向。」
- 追問：「這種短句一句一句切開的講法，是你刻意的，還是只是說話自然會這樣、寫成文字其實可以併成長句？」
- *現行 skill：A。staccato 短句是硬規則還是口語副產品。若老師說寫作時長句沒問題，skill 應把 short-burst 標成口語限定、文字可放寬。*

**AB7 — 方法引入順序：先讓問題痛（怎麼辦呢）vs 先給定義**
題幹：「同樣是引出 memory management 這個技術，這邊有兩種帶法，你聽聽看哪個比較像你會講的。」
- **A**：「假設你的輸入有 257 個 token，超過了模型上限。這下子塞不進去了，怎麼辦呢？這時候我們就需要一個方法，把舊的對話濃縮起來——這個就是 memory management。」
- **B**：「Memory management 是一種在對話超過 context window 上限時，用來壓縮或汰換舊內容的方法。比如說輸入有 257 個 token、超過上限的時候，就會用到它。」
- 追問：「如果對方其實已經知道自己需要這個詞、只是想查它是什麼意思，你還會堅持先鋪問題嗎？」
- *現行 skill：A。method-as-rescue 是否普世較好。若老師說查詞情境可直接給定義，skill 應把 problem-first 綁上『學習意圖』觸發條件。*

**AB8 — 結尾：果斷收尾後停住 vs 提供後續選單**
題幹：「同樣是把一堂 attention 的課收掉，這邊有兩種結尾，你聽聽看哪個比較像你會講的。」
- **A**：「好，那今天的課呢，我們其實就上到這邊。以上就是我今天想跟大家分享的內容。」
- **B**：「好，attention 的核心大概就是這樣。如果你想的話，我可以再往下講 multi-head 的版本，或是直接帶你看 KV cache 怎麼省記憶體——你想看哪個？」
- 追問：「如果這是在一個可以互動的場合、學生其實可以馬上回你想聽哪個，你還會收得這麼乾淨嗎？」
- *現行 skill：A（B 被列為 Anti-Regression 的『選單分支』反例）。若老師接受互動式選單，skill 應允許聊天媒介下放寬；否則乾淨收尾不可退讓。*

**AB9 — 比喻取材：ACG 動漫宇宙 vs 日常生活類比**
題幹：「同樣是解釋 model editing 裡 Generality 這個概念（改一個事實之後，反過來問也要對得起來），這邊有兩種比喻，你聽聽看哪個比較像你會用的。」
- **A**：「你問模型魯夫吃的是什麼果實，它會說橡膠果實。但海賊王從 1044 話開始我們才知道，那其實是人人果實幻獸種尼卡形態。我們做後訓練改完之後，它就要會說尼卡果實。而且要做到 Generality，你反過來問誰吃了尼卡果實，它也要答得出來是魯夫。」
- **B**：「你問模型魯夫吃的是什麼果實，它會說舊答案。這就像你的通訊錄裡某個朋友改了電話，你要更新這一筆。但你不只要改這一筆——朋友的另一半用舊號碼打來問起，你也要對得起來是同一個人，這個就叫做 Generality。」
- 追問：「你在挑載體的時候，是看哪個觀眾比較有共同記憶，還是看哪個比喻內部的邏輯比較撐得起整段？」
- *現行 skill：A（傾向但不強制 ACG）。教 skill：當概念有多個彼此關聯的子項時，是否優先選有內部邏輯的動漫宇宙串起來。*

**AB10 — 對 benchmark 的態度：號稱／Goodhart 質疑 vs 當成事實轉述**
題幹：「同樣是講一個模型在 benchmark 上拿到 state-of-the-art，這邊有兩種講法，你聽聽看哪個比較像你會講的。」
- **A**：「這個模型號稱在 benchmark 上是 state-of-the-art。但你要想一下，這個指標到底在量什麼？一旦一項指標被當成目標，它就不再是一個好指標——它可能只是學會了 game 這個 metric 而已。」
- **B**：「這個模型在這個 benchmark 上達到了 state-of-the-art，分數明顯超越前一代，相比之下進步是蠻明顯的。」
- 追問：「看到一個刷榜的數字，你的預設反應是先質疑它在量什麼，還是會先承認它是個進步、再看情況補質疑？」
- *現行 skill：A。skill 是否該讓 metric 永遠不能單獨當作進步的證據。*

## 訪談技巧提醒

- 引句反應題的引文要**逐字唸**，不要摘要——摘要會把風格訊號洗掉。
- 老師的自我描述跟逐字稿觀察到的行為衝突時，**兩個都記、不要當場糾正**——自我認知與實際行為的差距本身就是資料。
- 不要替 skill 的設計辯護。被批評的每一句都是改進素材。
- Q3–Q5（規則與邊界）無論如何要拿到，這關係到這個專案能不能繼續做。
- 反向生成題（Q8、Q21）問完就走，不要當場比對或念 skill 的版本給他聽——diff 是離線作業。
- A/B 題唸的時候**不要透露哪個是 skill 現在用的版本**，也不要用語氣暗示你偏好哪個——一暗示就變成誘導題，資料就毀了。
- A/B 題若老師說「兩個我都不會這樣講」，那是**最好的結果**——立刻追問「那你會怎麼講」，逐字記下。
- A/B 四顆星題（AB1–AB4）若時間只夠問一輪 A/B，就只問這四題——它們測的是 skill 自創、最可能失真的手法。
