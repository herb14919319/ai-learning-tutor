### 6.2 需要統一的名詞

#### Bias

- CH03 的 `Bias` 是偏差－變異權衡中的「模型系統性偏差／模型假設過度簡化」。
- CH06 的 `偏誤（Bias）` 是刻板印象、不公平或群體差異風險。

兩者英文相同但概念不同，學習者可能把高 Bias 欠擬合與公平性偏誤混為一談。建議全課程明確區分：

- **模型偏差（Model Bias）**：CH03 的 Bias－Variance 語境。
- **公平性偏誤（Social／Fairness Bias）**：CH06／CH07 的群體公平與刻板印象語境。

#### Human-in-the-loop

- CH01 將 HITL 嚴格定義為人類逐次參與關鍵決策。
- CH06／CH07 把 `Human-in-the-loop` 用作較廣義的人工覆核、能停止流程與重要節點審批。

後者包含 HITL，但也可能接近 CH01 的 HOTL。建議保留 CH01 的考試分類，後續章節改用「人工覆核／Human Oversight」作上位詞；只有人員確實位於決策路徑、未核准就不能繼續時，才標為 HITL。

#### Strong AI 與 AGI

CH01 已提醒不同文獻可能把 Strong AI 與 AGI 重疊使用，但核心概念、關鍵名詞與測驗仍呈現四層分類，容易讓學員誤以為所有資料都把兩者視為互斥層級。建議明示「本教材為應試分類；一般文獻常視兩者為近義或重疊」。

#### MCP、Tool Calling、AI Agent

- MCP 與 AI Agent 在 CH06 定義合理。
- 章文使用「函式／工具呼叫」，但沒有在首次出現時標示 `Tool Calling` 或 `Function Calling` 的英文名稱，也未列入關鍵名詞。
- `AI Agent` 使用「AI Agent」與「AI 代理」兩種形式；CH03 強化學習另有「代理人」，應避免讓學員誤以為兩者完全同義。

#### Prompt Injection

CH06 先出現「提示注入」，未附英文；到 CH07 才首次標示 `Prompt Injection`。依全課程閱讀順序，英文應在 CH06 第一次出現時標示。

#### No Code／No-Code

Metadata 與核心教材使用 `No Code / Low Code`，章名與內容使用 `No-Code／Low-Code`。兩種寫法不影響概念，但平台的正式顯示名稱與內文應選定一種格式；建議 metadata 正式章名保持原樣時，正文第一次出現可註記常見連字號寫法。
